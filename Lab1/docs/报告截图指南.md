## 6.1 实验场景(1)：仓库创建与提交

### R0：查看工作区状态
```bash
git status
```
**报告文字**：使用 `git status` 查看当前工作区、暂存区状态，确认各文件所处状态。

---

### R1：初始化仓库并加入源文件
```bash
git init
git add .
git status
```
**报告文字**：使用 `git init` 初始化本地 Git 仓库，使用 `git add .` 将 Lab1 项目的全部源文件纳入 Git 管理。

> ⚠️ 注意：由于仓库已经初始化过了，您可以直接截 `git log --oneline` 的第一条 `Initial commit of Lab1 project` 作为证据。

---

### R2：首次提交
```bash
git commit -m "Initial commit of Lab1 project"
```
**报告文字**：使用 `git commit` 将暂存区的所有文件提交到本地仓库，提交信息为 "Initial commit of Lab1 project"。

---

### R3：查看修改内容
```bash
# 先手动修改一个文件（比如在 README.md 末尾加一行文字），然后：
git status
git diff
```
**报告文字**：手工修改了 `README.md` 文件后，使用 `git status` 查看哪些文件被修改，使用 `git diff` 查看具体修改内容。

---

### R4：重新提交
```bash
git add .
git commit -m "Scene 1: First modification (R4)"
```
**报告文字**：将修改后的文件重新加入暂存区并提交，提交信息为 "Scene 1: First modification (R4)"。

---

### R5：再次修改并提交
```bash
# 再次修改文件后：
git add .
git commit -m "Scene 1: Second modification (R5)"
```
**报告文字**：再次对部分文件进行修改，重新 `git add` 并 `git commit` 提交。

---

### R6：撤销最后一次提交
```bash
git revert HEAD --no-edit
git log --oneline -n 5
```
**报告文字**：使用 `git revert HEAD` 撤销最后一次提交。`git log` 显示新增了一条 "Revert" 记录，证明撤销操作成功。

---

### R7：查询全部提交记录
```bash
git log --oneline
```
**报告文字**：使用 `git log` 查询该项目的全部提交记录，可以清晰看到从初始提交到撤销操作的完整历史。

**📸 截图**：截取完整的 `git log` 输出，或者直接截 IDE 的 Git Log 面板（您之前发的那张截图就可以用！）

---

### R8-R9：推送到 GitHub
```bash
git remote -v
git push origin 2023111320
```
**报告文字**：在 GitHub 上创建了名为 "Lab1-2023111320" 的仓库，使用 `git remote add origin` 关联远程仓库，并通过 `git push` 将本地仓库全部内容推送到 GitHub。

**📸 额外截图**：打开浏览器访问您的 GitHub 仓库页面，截取仓库首页（显示文件列表），证明推送成功。

## 6.2 实验场景(2)：分支管理

### R1：查看分支并切换到 master
```bash
git branch
git checkout master
```
**报告文字**：使用 `git branch` 获得本地仓库的全部分支，然后 `git checkout master` 切换至 master 分支。

---

### R2：创建 B1、B2 分支
```bash
git branch B1
git branch B2
git branch
```
**报告文字**：在 master 基础上使用 `git branch` 命令分别建立 B1、B2 两个分支。

---

### R3：在 B2 上创建 C4
```bash
git checkout B2
git branch C4
git checkout C4
```
**报告文字**：切换到 B2 分支，在其基础上创建新分支 C4，并切换到 C4。

---

### R4：在 C4 上修改 2 个文件并提交
```bash
# 修改 README.md 和 Easy Test.txt 后：
git add .
git commit -m "C4: Modify README and Easy Test"
```
**报告文字**：在 C4 分支上，分别对 `README.md` 和 `Easy Test.txt` 两个文件进行修改，然后 `git add` 和 `git commit` 提交。

---

### R5：在 B1 上修改同样文件并提交
```bash
git checkout B1
# 修改同样的 README.md 和 Easy Test.txt（做不同修改）后：
git add .
git commit -m "B1: Modify README and Easy Test"
```
**报告文字**：切换到 B1 分支，对同样的 `README.md` 和 `Easy Test.txt` 做不同的修改并提交，为后续合并冲突做准备。

---

### R6：将 C4 合并到 B1（触发冲突并手工消解）
```bash
git merge C4
# 出现冲突后，手动编辑冲突文件，解决冲突标记 <<<<<<< ======= >>>>>>>
git add .
git commit -m "Merged C4 into B1 and resolved conflict"
```
**报告文字**：在 B1 分支上执行 `git merge C4`，由于两个分支修改了同一文件的同一区域，产生了合并冲突。手工打开冲突文件，删除冲突标记并保留正确内容后，重新 `git add` 和 `git commit` 完成合并。

---

### R7：在 B2 上修改并提交
```bash
git checkout B2
# 修改 2 个文件后：
git add .
git commit -m "R7: B2 modifies 2 files"
```
**报告文字**：切换到 B2 分支，对 2 个文件进行修改并提交。

---

### R8：查看合并状态
```bash
git checkout 2023111320
git branch --merged
git branch --no-merged
```
**报告文字**：使用 `git branch --merged` 查看已经合并到当前分支的分支，使用 `git branch --no-merged` 查看尚未合并的分支。

---

### R9：删除已合并分支 + 合并未合并分支到学号分支
```bash
git branch -d C4
git merge B2 -m "Merge B2 into 2023111320"
```
**报告文字**：使用 `git branch -d` 删除已合并的分支 C4，然后将尚未合并的 B2 分支合并到以学号命名的分支 `2023111320` 上。

---

### R10：推送学号分支到 GitHub
```bash
git push origin 2023111320
```
**报告文字**：使用 `git push` 将本地的 `2023111320` 分支推送到 GitHub 远程仓库。

---

### R11：查看完整版本变迁树
```bash
git log --all --graph --oneline --decorate
```
**报告文字**：使用 `git log --all --graph --oneline --decorate` 查看完整的版本变迁树，可以清晰看到各分支的创建、合并和冲突解决过程。

**📸 截图**：直接使用您 IDE 的 Git Log 面板截图（之前那张带彩色线条的图）！

---

### R12：在 GitHub 上查看仓库
**📸 截图**：打开浏览器，进入 GitHub 仓库页面，截取：
1. 仓库首页（文件列表）
2. 分支下拉菜单（显示 2023111320 分支）

## 7. 在 IDE 中使用 Git Plugin

**报告文字**：

在 IDE（VS Code / PyCharm 等）中通过图形化界面进行 Git 操作：

### 7.1 将 Lab1 纳入 Git 管理
在 IDE 中打开项目文件夹 `软件工程lab`，IDE 自动识别 `.git` 目录，底部状态栏显示当前分支名 `2023111320`。

**📸 截图**：IDE 界面，底部状态栏显示分支名。

### 7.2 修改并提交
1. 在 IDE 中修改任意一个文件
2. 打开 Source Control 面板（VS Code 左侧栏）或 Git 面板（PyCharm：Alt+9）
3. 点击 "Stage All Changes"（暂存所有更改）
4. 在提交信息框输入提交说明
5. 点击 "Commit" 按钮完成提交

**📸 截图**：
- 截取 Source Control / Git 面板中显示已修改文件的界面
- 截取填写提交信息并点击 Commit 的界面

### 7.3 推送到 GitHub
1. 在 IDE 中点击 "Push" 按钮（或使用快捷键）
2. 确认推送到远程仓库 `origin/2023111320`

**📸 截图**：
- 截取推送操作的界面或推送成功的通知
- 截取 IDE 的 Git Log 面板，显示完整的提交历史树
